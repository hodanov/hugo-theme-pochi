# Plan: Modern blog feature ideas

pochi テーマに取り入れられそうな 2025-2026 年のブログデザイン・機能トレンドを整理したアイディア集。実装するかどうかは未定。優先度検討やロードマップ策定のインプットとして使う。

## Background

- pochi テーマには既にダークモード、検索、TOC、Giscus コメント、関連記事、いいね、シェア、Lightbox など主要機能が揃っている
- 2025-2026 のブログトレンド（マイクロインタラクション、Bento Grid、View Transitions、コードコピーボタンなど）と比較して、差分となる機能を洗い出した
- 日本語の技術ブログ（Zenn, Qiita など）で標準化している機能も考慮

## Current structure

pochi テーマの既存機能:

- ダークモード切り替え（OS 連動 + 手動トグル + localStorage 永続化）
- クライアントサイド検索（Fuse.js + Mark.js）
- TOC サイドバー + ScrollSpy
- コメント（Giscus / Disqus）
- 関連記事（タグベース、3 件表示）
- いいねボタン（API 連携 + localStorage）
- シェアボタン（URL コピー / Web Share API）
- パンくずリスト
- Lightbox（画像拡大表示）
- 言語切り替え（日本語 / 英語）
- レスポンシブ対応 + モバイルフローティングアクションバー
- アーカイブページ（年月折りたたみ）
- スムーススクロール + スクロールトップボタン
- Google AdSense 対応
- Schema.org 構造化データ

## Feature ideas

### 1. コードブロックのコピーボタン

- 技術ブログでは事実上の標準機能（Zenn, Qiita, dev.to 等で実装済み）
- `<pre><code>` ブロックの右上にコピーボタンを配置
- Clipboard API でコピー、フィードバック表示（「コピーしました」）

| 項目     | 内容                     |
| -------- | ------------------------ |
| 難易度   | 低                       |
| 変更範囲 | JS 1 ファイル + CSS 追加 |
| 依存     | なし                     |

### 2. 記事の推定読了時間

- 記事冒頭に「約 X 分で読めます」を表示
- Hugo の `.ReadingTime` で取得可能
- `post_meta_date.html` または新規 partial で表示

| 項目     | 内容                                  |
| -------- | ------------------------------------- |
| 難易度   | 低                                    |
| 変更範囲 | テンプレート 1 行追加 + i18n キー追加 |
| 依存     | なし                                  |

### 3. 読了プログレスバー

- ページ上部に固定表示するスクロール連動のプログレスバー
- 記事をどこまで読んだか視覚的に示す
- CSS + 数行の JS で実装可能

| 項目     | 内容                                                       |
| -------- | ---------------------------------------------------------- |
| 難易度   | 低                                                         |
| 変更範囲 | JS 追加 + CSS 追加 + baseof.html or single.html に要素追加 |
| 依存     | なし                                                       |

### 4. SNS シェアボタンの拡充

- 現在は URL コピーのみ
- 追加候補: X (Twitter)、はてなブックマーク、Pocket
- 特にはてブは日本語の技術ブログでは必須級
- intent URL ベースのため外部ライブラリ不要

| 項目     | 内容                                               |
| -------- | -------------------------------------------------- |
| 難易度   | 低〜中                                             |
| 変更範囲 | `share_button.html` 拡張 + アイコン SVG 追加 + CSS |
| 依存     | なし                                               |

### 5. Callout / Admonition（注意書きブロック）

- `info`, `warning`, `tip`, `danger` などの装飾ブロック
- 実装方法の選択肢:
  1. Hugo v0.134+ の Obsidian 風 Blockquote Alerts（`> [!NOTE]` 記法）
  2. カスタム shortcode
- 技術記事の可読性が大きく向上する

| 項目     | 内容                                              |
| -------- | ------------------------------------------------- |
| 難易度   | 中                                                |
| 変更範囲 | shortcode or render-blockquote テンプレート + CSS |
| 依存     | Hugo v0.134+ (Blockquote Alerts の場合)           |

### 6. View Transitions API

- ページ遷移時のスムーズなアニメーション
- `<meta name="view-transition" content="same-origin">` で有効化
- Chrome 系ブラウザで対応済み、Safari は段階的に対応中
- サイト全体が SPA 風の体験になる

| 項目     | 内容                                                    |
| -------- | ------------------------------------------------------- |
| 難易度   | 中                                                      |
| 変更範囲 | `head.html` に meta タグ追加 + CSS でトランジション定義 |
| 依存     | ブラウザ対応状況（プログレッシブエンハンスメント前提）  |

### 7. 記事シリーズナビゲーション

- 連載記事を「前の記事 / 次の記事」でナビゲート
- Hugo の taxonomy で `series` を追加して管理
- シリーズ一覧ページも自動生成可能

| 項目     | 内容                               |
| -------- | ---------------------------------- |
| 難易度   | 中                                 |
| 変更範囲 | taxonomy 設定 + partial 追加 + CSS |
| 依存     | なし                               |

### 8. スクロールトリガーのフェードインアニメーション

- 記事カードやセクションが画面に入ったときにふわっと表示
- `IntersectionObserver` ベースで軽量実装
- 2025-2026 のマイクロインタラクショントレンドの王道
- パフォーマンスへの影響を最小限に（CSS transform + opacity のみ）

| 項目     | 内容                                               |
| -------- | -------------------------------------------------- |
| 難易度   | 中                                                 |
| 変更範囲 | JS モジュール追加 + CSS + テンプレートにクラス付与 |
| 依存     | なし                                               |

### 9. Bento Grid レイアウト（ホームページ）

- トップページを弁当箱風のモジュラーグリッドレイアウトに再設計
- 最新記事、人気記事、カテゴリ、プロフィールなどをブロック配置
- CSS Grid ベースで実装
- 日本のデザイントレンドとも相性が良い

| 項目     | 内容                                                      |
| -------- | --------------------------------------------------------- |
| 難易度   | 高                                                        |
| 変更範囲 | ホームページレイアウト再設計 + CSS Grid + 新規 partial 群 |
| 依存     | なし                                                      |

### 10. 自前コメントシステム

- Giscus の代替として独自コメントシステムを構築
- プライバシー重視のトレンドと合致
- 既に `docs/plan/2026-03-28_custom-comment-system.md` でプラン策定済み

| 項目     | 内容                                                |
| -------- | --------------------------------------------------- |
| 難易度   | 高                                                  |
| 変更範囲 | バックエンド API + フロントエンド UI + テンプレート |
| 依存     | 外部 API サーバー                                   |

## Prioritization guide

コストパフォーマンス順の推奨グループ:

1. **すぐやれる（低コスト・高効果）**: コピーボタン、読了時間、プログレスバー、SNS シェア拡充
2. **中期的に取り組む（中コスト・差別化）**: Callout、View Transitions、シリーズナビ、フェードインアニメーション
3. **大きめの取り組み（高コスト・体験刷新）**: Bento Grid、自前コメント

## Open questions

- Bento Grid 導入時、既存のリスト表示との共存 or 完全置換か
- View Transitions の Safari 対応状況を待つかどうか
- フェードインアニメーションの `prefers-reduced-motion` 対応方針
