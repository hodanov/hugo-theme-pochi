# Plan: レガシーコードのクリーンアップと CSS/HTML の正規化

非推奨の OGP prefix 属性、IE 向けルール、CSS 重複宣言を削除し、ダークモードの CSS フォールバックと lang_switcher のインライン JS 外部化を行う。

## Background

- フェーズ 1〜3 に続く最終フェーズとして、影響が小さいクリーンアップを実施する
- いずれも機能的な問題は軽微だが、保守性や HTML バリデーション、CSP 準拠に関わる改善
- 各変更は独立しているため、個別にコミットしても一括でコミットしても問題ない

## Current structure

- `layouts/_default/baseof.html:9-11` — `<head prefix="og: http://ogp.me/ns# fb: http://ogp.me/ns/fb# ...">` 非推奨の RDFa 名前空間
- `assets/css/main.css:329-332` — `/* IE用の設定 */ a img { border: none; }` IE 向けルール
- `assets/css/base.css:14-16` — `body { font-family: var(--pochi-font-base); }` main.css と重複
- `layouts/partials/molecules/lang_switcher.html:20` — `onchange="..."` インライン JS
- `assets/css/tokens.css:79-99` — `:root.dark` のみ。`@media (prefers-color-scheme: dark)` なし
- `layouts/partials/head/head.html:50-65` — ダークモード初期化のインラインスクリプト

## Implementation steps

1. **`baseof.html` の `<head prefix>` 属性を削除** — `<head>` からRDFa の prefix 属性を削除する。OGP の `<meta property="og:...">` は prefix なしで全クローラーに認識される
2. **IE 向け CSS ルールを削除** — `main.css` の `/* IE用の設定 */ a img { border: none; }` を削除する。modern-normalize で対応済みであり IE はサポート終了済み
3. **`base.css` の `body { font-family }` 重複宣言を削除** — `main.css:12` に同じ宣言があるため `base.css:14-16` の `body { font-family: var(--pochi-font-base); }` を削除する
4. **`lang_switcher.html` のインライン JS を外部化** — `onchange` 属性を削除し、新規 `assets/js/lang-switcher.js` にイベントリスナーを登録する。`select` 要素に `data-pochi-lang-switcher` 属性を追加して JS 側で取得する。PJAX 対応としてイベント委譲（`document.addEventListener("change", ...)` でセレクタをフィルタ）を使用する。`pochi:afterSwap` イベントでの再バインドは不要
5. **`tokens.css` に `@media (prefers-color-scheme: dark)` ブロックを追加** — `:root.dark` と同じ変数を `@media (prefers-color-scheme: dark) { :root:not(.light) { ... } }` にも追加する。JS 無効環境でも OS 設定に追従するようになる
6. **テーマ切替 JS に `.light` クラスの付与を追加** — `main.js` のテーマ切替ロジックで、ライトモード選択時に `root.classList.add("light")` を、ダークモード選択時に `root.classList.remove("light")` を実行する。`head.html` の FOUC 防止スクリプトでも `pref-theme === "light"` 時に `.light` クラスを付与する。これにより CSS の `@media (prefers-color-scheme: dark) { :root:not(.light) }` がユーザーの明示的なライト選択を尊重する

## Risks and mitigations

| Risk                                                                                                         | Mitigation                                                                                                                                                                 |
| ------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `<head prefix>` 削除で OGP が認識されなくなる                                                                | Facebook, Twitter, その他のクローラーは prefix 属性なしで `<meta property="og:...">` を認識する。Hugo の `_internal/opengraph.html` テンプレートも prefix に依存していない |
| `a img { border: none; }` 削除で画像リンクにボーダーが出る                                                   | modern-normalize で `img { border-style: none; }` が適用されているため影響なし                                                                                             |
| `base.css` の `body { font-family }` 削除で他のプロパティが消える                                            | 当該ブロックは `font-family` の 1 プロパティのみなので、ブロック全体を削除しても他への影響なし                                                                             |
| lang_switcher の JS 外部化で PJAX ナビゲーション後に動作しなくなる                                           | イベント委譲（`document.addEventListener("change", ...)`）を使用するため、DOM の差し替え後も自動的に動作する                                                               |
| `@media (prefers-color-scheme: dark)` 追加で、ユーザーが明示的にライトモードを選択した後もダークモードに戻る | CSS セレクタを `:root:not(.light)` に限定し、JS でライト選択時に `.light` クラスを付与する。これにより media query よりユーザーの明示的な選択が優先される                  |
| `.light` クラス追加で既存のテーマ切替ロジックが複雑化する                                                    | `.light` は CSS の media query 無効化のみに使用し、テーマ状態の判定は従来通り `.dark` の有無で行う。`.light` はダークモードの media query を抑制するためだけの補助クラス   |
